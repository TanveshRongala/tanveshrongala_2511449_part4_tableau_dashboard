# Chart Selection Justification

| Dashboard View | Business Question Answered | Chart Type and Why It Fits | Fields Used for Color / Size / Label / Filter | Design Principle Applied | Mistake Avoided |
|---|---|---|---|---|---|
| Sales Trend View | How are sales changing over time? | Line chart because time-series movement is best read as a continuous trend. | Date on x-axis, sales on y-axis, latest month annotation. Date filter available. | Clear hierarchy and readable time labels. | Avoided pie charts for time-series data. |
| Regional Performance View | Which region performs best? | Clustered bar chart because leaders can compare sales and profit side by side by region. | Region as dimension; sales and profit as measures; region filter/action. | Sorted bars and consistent business colors. | Avoided unsorted categories and decorative 3D charts. |
| Category Profitability View | Which categories/sub-categories drive profit or losses? | Horizontal bar chart because it supports long sub-category labels and clearly shows positive vs negative profit. | Sub-category labels; profit values; color indicates profitable vs loss-making. | Direct labeling and zero baseline. | Avoided misleading axis truncation. |
| Customer Segment View | Which segments are valuable or risky? | Bar comparison because segment-level margin and return rate need quick side-by-side comparison. | Segment dimension; profit margin and return rate measures; segment filter. | Balanced risk and value view. | Avoided showing only sales without quality/risk metrics. |
| Shipping Performance View | Which ship mode causes delay? | Bar chart because average delivery days can be compared directly across shipping modes. | Ship mode on x-axis; average delivery days on y-axis; ship mode filter. | Simple operational metric with clear title. | Avoided overcomplicated maps or scatter plots. |
| Discount vs Profit View | How does discount relate to profit? | Bubble/scatter-style chart because the relationship between discount band, margin, and order volume needs comparison. | Discount band on x-position/label; profit margin on y-axis; order count by size. | Encodes three variables without clutter. | Avoided using a pie chart for relationship analysis. |
| Return Analysis View | Which categories or regions have return risk? | Bar chart because return rates by group need ranked comparison. | Category/region dimension; return rate measure; return flag calculation. | Risk-focused color and percentage labels. | Avoided counting returns without normalizing by order volume. |
| Executive Dashboard | What opportunities and risks should leadership act on? | Combined KPI cards and charts because leaders need both summary and drill-down context. | Filters: Region, Category, Customer Segment, Date, Ship Mode, Campaign Channel; action: select Region to filter views. | Clear top-down layout: KPIs first, trends/comparison second, diagnostics third. | Avoided a random chart collection with no business story. |
